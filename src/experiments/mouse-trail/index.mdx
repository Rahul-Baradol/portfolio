## Starting off with naive implementation

What do you expect ?

What all things we need?
- How to keep track of images in action ?
- How to animate them ?

We can define each Image with following properties

```jsx
interface Image {
    x: number, // current x position
    y: number, // current y position
    dx: number, // change in x position
    dy: number, // change in y position
    id: number, // unique identifier
    src: string, // image source
    stage: 'initial-bounce-up' | 'free-fall' | 'bounce-up' | 'bounce-down'; // stage of the animation
} 
```

And maintain a state of these **images**

```jsx
const [images, setImages] = useState<Image[]>([]);
```

And then for each mouse event, we would "spawn" images by populating `images` state

```jsx
const handleMouseMove = (event: React.MouseEvent) => {
    const container = containerRef.current;
    if (!container) return;

    const rect = container.getBoundingClientRect();
    const curX = event.clientX - rect.left;
    const curY = event.clientY - rect.top;

    const now = performance.now();
    if (now - lastSpawnTime.current < 100) {
        return;
    }

    lastSpawnTime.current = now;

    const randomImageIndex = Math.floor(Math.random() * imageLinks.length); 

    const newImage: Image = {
        x: curX - 50,
        y: curY - 50,
        id: imageLinks[randomImageIndex].id,
        src: imageLinks[randomImageIndex].src,
        stage: 'initial-bounce-up',
        dx: (Math.random() * 10) - 5,
        dy: 10
    };

    setImages(prevImages => {
        return [...prevImages, newImage];
    });
}
```

And we start a render loop like this

```jsx
useEffect(() => {
    let animationFrameId: number;
    const renderLoop = () => {
        setImages(prevImages => {
            if (prevImages.length === 0) {
                return [];
            }

            let updatedImages = prevImages.map(image => {
                const newImage = {
                    ...image,
                }

                if (image.stage === 'initial-bounce-up') {
                    newImage.y -= newImage.dy;
                    newImage.x += newImage.dx;

                    newImage.dy -= 1;
                    newImage.dx *= 0.98;
                    if (newImage.dy <= 0) {
                        newImage.stage = 'free-fall';
                        newImage.dy = 5;
                    }
                } else if (image.stage === 'free-fall') {
                    newImage.y += newImage.dy;
                    newImage.dy += 1;

                    if (newImage.y >= (window.innerHeight - 100)) {
                        newImage.stage = 'bounce-up';
                        newImage.dy = 10;
                    }
                } else if (image.stage === 'bounce-up') {
                    newImage.y -= newImage.dy;
                    newImage.dy -= 1;
                    if (newImage.dy <= 0) {
                        newImage.stage = 'bounce-down';
                        newImage.dy = 5;
                    }
                } else if (image.stage === 'bounce-down') {
                    newImage.y += newImage.dy;
                    newImage.dy += 1;
                }

                return newImage;
            });

            updatedImages = updatedImages.filter(image => image.y < window.innerHeight);

            return updatedImages;
        });

        animationFrameId = requestAnimationFrame(renderLoop);
    }

    animationFrameId = requestAnimationFrame(renderLoop);

    return () => {
        cancelAnimationFrame(animationFrameId);
    }
}, []);
```

And we render these as follows

```jsx

))}
```

Now before we get into optimizations, let's take a look at the current implementation and identify the bottlenecks.

1. **Frequent Layout Calculations**: The current implementation triggers layout calculations on every frame, which is expensive. This is especially true for the "Layout" stage, which can be a significant performance bottleneck.

2. **Image Management**: The way images are managed and updated in the state can lead to unnecessary re-renders. Each image's position is updated individually, which can be optimized.

3. **Rendering Pipeline**: The naive approach goes through the entire rendering pipeline (DOM > Style Calculation > Layout > Paint > Composite) for each frame, leading to potential performance issues.

To address repeated-layout-recalculation, let's use `transforms` instead of `top` or `left`. 

Because you might know, that `transforms` only trigger composition which is much cheaper as compared to full layout recalculation, and this I am talking about every frame!

So the next phase would be where we optimize the way we store and manage the images in the state.
Let's not use React "state" at all, and modify the DOM directly, bypassing React's reconciliation.

Why?

Well React is not good for high-frequency updates like animations.

So how to do it?

We maintain the images as a React ref, which doesn't trigger a React re-render on update, and we directly manipulate the DOM to update the positions of the images.

```jsx
const images = useRef<Image[]>([]);
```

And for each mouse move, we update the ref, which doesn't trigger a re-render 
```jsx
const handleMouseMove = (event: React.MouseEvent) => {
    if (!paintElement.current) {
        return;
    }
    const rect = paintElement.current.getBoundingClientRect();
    const curX = event.clientX - rect.left;
    const curY = event.clientY - rect.top;

    const now = performance.now();
    if (now - lastSpawnTime.current < 100) {
        return;
    }

    lastSpawnTime.current = now;

    const randomImageIndex = Math.floor(Math.random() * imageLinks.length); 

    const newImage: Image = {
        x: curX - 50,
        y: curY - 50,
        id: imageLinks[randomImageIndex].id,
        domId: Math.floor(Math.random() * 10000), 
        src: imageLinks[randomImageIndex].src,
        stage: 'initial-bounce-up',
        dx: (Math.random() * 10) - 5,
        dy: 10,
        insertedInDom: false,
        shouldBeDeleted: false,
    };

    images.current.push(newImage);
}
```

And the render loop slightly changes as follows
```jsx
useEffect(() => {

                } else if (image.stage === 'free-fall') {
                    image.y += image.dy;
                    image.dy += 1;

                    if (image.y >= (window.innerHeight - 100)) {
                        image.stage = 'bounce-up';
                        image.dy = 10;
                    }
                } else if (image.stage === 'bounce-up') {
                    image.y -= image.dy;
                    image.dy -= 1;
                    if (image.dy <= 0) {
                        image.stage = 'bounce-down';
                        image.dy = 5;
                    }
                } else if (image.stage === 'bounce-down') {
                    image.y += image.dy;
                    image.dy += 1;
                }

                if (image.insertedInDom === false) {
                    const domImage = document.createElement('img');
                    domImage.id = `image-${image.domId}`;
                    domImage.src = image.src;
                    domImage.style.position = 'absolute';
                    domImage.style.width = '100px';
                    domImage.style.height = '100px';
                    domImage.style.left = `${image.x}px`;
                    domImage.style.top = `${image.y}px`;
                    paintElement.current!.appendChild(domImage);

                    imageCache.current.set(image.domId, domImage);

                    image.insertedInDom = true;
                } else if (image.y < window.innerHeight) {
                    const domImage = imageCache.current.get(image.domId);
                    if (domImage) {
                        domImage.style.left = `${image.x}px`;
                        domImage.style.top = `${image.y}px`;
                    }
                } else {
                    const domImage = imageCache.current.get(image.domId);
                    if (domImage) {
                        paintElement.current!.removeChild(domImage);
                        imageCache.current.delete(image.domId);
                        image.shouldBeDeleted = true;
                    }
                }
            });

            images.current = images.current.filter(image => !image.shouldBeDeleted);

            animationFrameId = requestAnimationFrame(renderLoop);
        }

        animationFrameId = requestAnimationFrame(renderLoop);

        return () => {
            cancelAnimationFrame(animationFrameId);
        }
    }
}, [paintElement]);
```

Can we make it even better?

Yes!

Moving to canvas.

The reason is because, in canvas approach we would handle the rendering ourselves, giving us more control over the performance.

That also means, we have to be careful in memory handling, making sure we don't create GC pressure.

In canvas approach, we would still store images as a React ref, handle the mouse event the same way as before, but the way we handle rendering would change significantly. Instead of relying on the DOM and React's reconciliation, we would draw the images directly onto the canvas.

The render loops changes to the following
```jsx
const renderLoop = () => {
    let updatedImages = images.current.map(image => {
        let newImage = { ...image };

        if (newImage.stage === 'initial-bounce-up') {
            newImage.y -= newImage.dy;
            newImage.x += newImage.dx;

            newImage.dx *= 0.98;
            newImage.dy -= 1;

            if (newImage.dy <= 0) {
                newImage.stage = 'free-fall';
                newImage.dy = 5;
            }
        } else if (newImage.stage === 'free-fall') {
            newImage.y += newImage.dy;
            newImage.dy += 1;

            if (newImage.y >= (window.innerHeight - 100)) {
                newImage.stage = 'bounce-up';
                newImage.dy = 10;
            }
        } else if (newImage.stage === 'bounce-up') {
            newImage.y -= newImage.dy;
            newImage.dy -= 1;

            if (newImage.dy <= 0) {
                newImage.stage = 'bounce-down';
                newImage.dy = 5;
            }
        } else if (newImage.stage === 'bounce-down') {
            newImage.y += newImage.dy;
            newImage.dy += 1;
        }

        return newImage;
    });

    updatedImages = updatedImages.filter(image => image.y < window.innerHeight);

    images.current = updatedImages;

    const canvas = canvasRef.current;
    if (!canvas) {
        return;
    }

    const ctx = canvas.getContext("2d");
    if (!ctx) {
        return;
    }

    ctx.clearRect(0, 0, canvas.width, canvas.height);

    updatedImages.forEach(image => {
        const img = imageCache.current.get(image.src);
        if (img) {
            ctx.drawImage(img, image.x, image.y, 100, 100);
        }
    });

    animationFrameRef.current = requestAnimationFrame(renderLoop);
}
```

Few interesting lines worth mentioning are 
```jsx
updatedImages.forEach(image => {
    const img = imageCache.current.get(image.src); // what this !?
    if (img) {
        ctx.drawImage(img, image.x, image.y, 100, 100);
    }
});
```

What is this?

Well traditionally, we would think of doing something like this
```jsx
updatedImages.forEach(image => {
    const img = new Image();
    img.src = image.src;
    ctx.drawImage(img, image.x, image.y, 100, 100);
});
```

The above approach is bad for performance.
We are creating a new image every frame, right?
The older images are not being cleared from the tab memory, which increases memory usage.

When a GC is triggered, rendering is stopped for a brief moment, until the garbage collection completes.
If the stale image nodes are "too many", like 10K which is easily possible, the GC would take a significant amount of time to clean up, leading to noticeable frame drops and a poor user experience.

So what we do is during initial mount of component, we create images, one for each image link. These images are then cached in a `Map` for quick access during the render loop.
```jsx
useEffect(() => {
    const canvas = canvasRef.current;
    if (!canvas || !containerRef.current) {
        return;
    }

    imageLinks.forEach(link => {
        if (!imageCache.current.has(link.src)) {
            const img = new Image();
            img.src = link.src;
            imageCache.current.set(link.src, img);
        }
    });

    const handleWindowResize = () => {
        ...
    }

    handleWindowResize();
    animationFrameRef.current = requestAnimationFrame(renderLoop);

    window.addEventListener("resize", handleWindowResize);

    return () => {
        if (animationFrameRef.current) {
            cancelAnimationFrame(animationFrameRef.current);
        }

        window.removeEventListener("resize", handleWindowResize);
    }
}, []);
```

One small problem i noticed when opened on my Dell laptop, the images are falling twice as much as they should be.
To the point, where it is not visually appealing anymore.

Then I realised it is the frame-rate!

Mac's refresh rate is 60Hz, and the Dell laptop's refresh rate is 120Hz, and our logic of updating the image position is frame-rate dependent, and not time dependent!

For example, the following line 
```jsx
newImage.y += newImage.dy;
```

would run 60 times on a Mac and 120 times on a Dell laptop, in a second...

If `newImage.dy` is 1 let's say, then it `newImage.y` would be increased by 60px on a Mac, and 120px on a Dell laptop, in one second, where ideally we would need them to be the same.

We need to fix this!

We start with the following line 
```jsx
const lastFrameTime = useRef<number | null>(null);
```

And the render loop would be 
```jsx
const renderLoop = () => {
    const now = performance.now();
    let frameFactor = lastFrameTime.current ? (now - lastFrameTime.current) / 16.666 : 1;
    frameFactor = Math.min(frameFactor, 2);
    lastFrameTime.current = now;

    let updatedImages = images.current.map(image => {
        let newImage = { ...image };

        if (newImage.stage === 'initial-bounce-up') {
            newImage.y -= newImage.dy * frameFactor;
            newImage.x += newImage.dx * frameFactor;

            newImage.dx *= Math.pow(0.98, frameFactor);
            newImage.dy -= 1 * frameFactor;

            if (newImage.dy <= 0) {
                newImage.stage = 'free-fall';
                newImage.dy = 5;
            }
        } else if (newImage.stage === 'free-fall') {
            newImage.y += newImage.dy * frameFactor;
            newImage.dy += 1 * frameFactor;

            if (newImage.y >= (window.innerHeight - 100)) {
                newImage.stage = 'bounce-up';
                newImage.dy = 10;
            }
        } else if (newImage.stage === 'bounce-up') {
            newImage.y -= newImage.dy * frameFactor;
            newImage.dy -= 1 * frameFactor;

            if (newImage.dy <= 0) {
                newImage.stage = 'bounce-down';
                newImage.dy = 5;
            }
        } else if (newImage.stage === 'bounce-down') {
            newImage.y += newImage.dy * frameFactor;
            newImage.dy += 1 * frameFactor;
        }

        return newImage;
    });

    updatedImages = updatedImages.filter(image => image.y < window.innerHeight);

    images.current = updatedImages;

    const canvas = canvasRef.current;
    if (!canvas) {
        return;
    }

    const ctx = canvas.getContext("2d");
    if (!ctx) {
        return;
    }

    ctx.clearRect(0, 0, canvas.width, canvas.height);

    updatedImages.forEach(image => {
        const img = imageCache.current.get(image.src);
        if (img) {
            ctx.drawImage(img, image.x, image.y, 100, 100);
        }
    });

    animationFrameRef.current = requestAnimationFrame(renderLoop);
};
```

where we enforce 60Hz by adding this `frameFactor` 

Also we do 
```jsx
frameFactor = Math.min(frameFactor, 2);
```
to prevent overshooting of the frameFactor. Which could happen when the user switches tabs, or moves away from the window, leading to a high frameFactor ( because we compare the lastFrameTime and currentFrameTime )
